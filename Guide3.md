# Exercise 03: Schedule Automated Backup Task with Cron

### Estimated Duration: 0.5 Hour(s)

## Scenario

You are continuing work as the cloud automation engineer for this environment. A recurring file backup process must be configured on the Linux jump VM so that important source content is copied into a backup location automatically. Your task is to make sure the required script is ready to run, writes backup data to the correct destination, and is scheduled to execute on the required recurring interval.

## Overview

In this exercise, you will review and complete the Linux backup automation on the jump VM. You will work with the required backup script path, confirm that backups are written to the expected destination folder, and configure the correct cron schedule so the task runs automatically every five minutes.

## Objectives

- Task 1: Validate the backup script configuration
- Task 2: Configure the recurring cron schedule
- Task 3: Confirm backup output is created

## Task 1: Validate the backup script configuration

In this task, you will assess the backup automation requirements on the Linux jump VM and ensure the script is prepared for scheduled execution.

1. Sign in to the Azure portal at <https://portal.azure.com> using **Username:** `<inject key="AzureAdUserEmail"></inject>` and **Password:** `<inject key="AzureAdUserPassword"></inject>`, then connect to the jump VM associated with deployment **<inject key="DeploymentID" enableCopy="false"></inject>**.
2. Verify that the backup script is present at **`/opt/cloudlabs/scripts/backup.sh`**.
3. Ensure the script is suitable for non-interactive execution and that it writes backup content to **`/opt/cloudlabs/backups`**.
4. Confirm the script can be executed by the scheduled task mechanism.

> [!Important]
> The required script path for this exercise is **`/opt/cloudlabs/scripts/backup.sh`**, and the required backup destination is **`/opt/cloudlabs/backups`**.

## Task 2: Configure the recurring cron schedule

In this task, you will configure the Linux scheduler so the backup job runs at the required interval.

1. Create or update the appropriate cron configuration for the user context used in the lab.
2. Schedule **`/opt/cloudlabs/scripts/backup.sh`** to run with the exact cron expression **`*/5 * * * *`**.
3. Ensure the scheduled entry is saved correctly and can persist for repeated execution during the lab.

> [!Note]
> The expected cron schedule for validation is **`*/5 * * * *`**.
